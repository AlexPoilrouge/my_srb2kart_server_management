About Discord Strashbot

Discord StrashBot is my personal Discord bot.
Depending on which guild you meet him, its function can differ.

As of ver. 0.3.0 there are mainly 2 usage for strashBot:

  • Smash Tool: offering few services for the Smash Strasbourg's community.
  • SRB2Kart Server Manager: tool to manage the StrashBot SRB2Kart Server.

Current version

Bot Version 0.3.1.

About slash commands

Since version 0.3.0 the Discord StrashBot transited from custom !commands to integrated slash commands

Technology

Discord Strashbot is build using node.js and Discord.js

Discord Strashbot's code (as most of its code) is available on my github.

/kart_info

The /kart_info command allows to check the current state of the StrashBot Karting SRB2Kart server. This give the user info about the number of player currently on the server, the map they're racing on, and other various info such as kartspeed, current gamemode, etc.

This command is basically a call to the /service and /info entrypoints of the strashbot.fr /kart API.

/kart_addons

The /kart_addons commands are for info about the addons currently installed on the StrashBot Karting SRB2Kart server.

/kart_addons list

Use the /kart_addons list subcommand to list the addons currently installed and running on the SRB2Kart Server.

Options:

  • pattern - give a pattern to seach for among installed addon. For example, /kart_addons list pattern:strash lists all the available addons containing the string "strash" in their name.

3 categories are displayed:

  • [Pending] - for addons that are going to be installed on the server's next restart.
  • [Downloaded] - for addons that are currently installed on the server.
  • [Base] - addons that are un-removable.

/kart_addons zip

Gives a link to a dowloadable .zip archive containing all the addons.

/kart_ingames

/kart_ingames is a group of commands used to check the assets available on the server. It differs for /kart_addons since it allows to check items in term of ingame objects, in this case racers and maps, instead of purely addon files.

/kart_ingames maps

Use the /kart_ingames maps subcommand to list the maps available on the server.

Options:

  • pattern - a string that is a part of the name of the map to look for. Example: /kart_ingames maps pattern:beach
  • type - specify if you want to list non-standard map. Possible choices:
    • Battle maps - to list battle maps
    • Hell maps - to list hell maps
    • Banned maps - to list maps that are installed but banned
  • sections - specify if you want to list section maps or not. Possible choices:
    • Show all maps (default) - list both lap maps and section maps
    • Only show section maps - list only section maps
    • Don't show section maps - list only lap maps

/kart_ingames map_count

/kart_ingames map_count is basically the same command as /kart_ingames maps, but it just counts the number of maps.
Options are identical.

Options:

  • pattern - a string that is a part of the name of the map to look for. Example: /kart_ingames map_count pattern:beach
  • type - specify if you want to count non-standard map. Possible choices:
    • Battle maps - to count battle maps
    • Hell maps - to count hell maps
    • Banned maps - to count maps that are installed but banned
  • sections - specify if you want to count section maps or not. Possible choices:
    • Show all maps (default) - counts both lap maps and section maps
    • Only show section maps - counts only section maps
    • Don't show section maps - counts only lap maps

/kart_ingames racers

Use the /kart_ingames racers subcommand to list the racers/characters/skins available on the server.

Options:

  • pattern - a string that is a part of the name of the racers name to look for. Example: /kart_ingames racer_count pattern:sonic
  • speed - a int to list every racer that has their speed stat set to this amount
  • weight - a int to list every racer that has their weight stat set to this amount

/kart_ingames racer_count

/kart_ingames racers is basically the same command as /kart_ingames maps, but it just counts the number of racers/characters/skins available.
Options are identical.

Options:

  • pattern - a string that is a part of the name of the racers name to look for. Example: /kart_ingames racers pattern:sonic
  • speed - a int to count the number of racer that has their speed stat set to this amount
  • weight - a int to count the number of racer that has their weight stat set to this amount

/kart_clips

The /kart_clips helps to manage some of the player cool clips by referencing them in the strashbot.fr's strashtheque.
Clips must be either .mp4/.ogg/.webm/.gif video uploaded via Discord, or youtube video links.

/kart_clips here

Use the /kart_clips here subcommand to register a clip url to the database. You can either pass the clip as a youtube url, or a video (or .gif) file through Discord (see available options).

Options:

  • clip_file - use this option to upload a clip file through Discord (must be .mp4/.ogg/.webm/.gif). (This option is prioritized over clip_url.)
  • clip_url - use this option to register a new youtube video link as a clip. (The clip_file option is prioritized over this one.)
  • description - a short description of your clip.

/kart_clips info

The /kart_clips info subcommand fetches infos about the strashtheque clips, or a clip in particular given its strashtheque ID.

Options:

  • clip_id - give info about a clip bearing the given ID (a number available on each clip of the strashtheque)

/kart_clips remove

The /kart_clips remove subcommand allows a user to remove a clip from the strashtheque.
Note: only a user that uploaded/registered a clip can remove it.

Options:

  • clip_id [Required]- the clip to remove's strashtheque ID.

/kart_clips description

The /kart_clips description subcommand allows a user to edit a clip from the strashtheque's description.
Note: only a user that uploaded/registered a clip can edit its description.

Options:

  • clip_id [Required]- the clip's strashtheque ID.
  • description - the description string of the clip (if none is passed, the clip's description will be deleted)

Kart Slash Commands [Restricted access]

/kart_password

The kart /kart_password command allows for an authorized user to fetch the StrasBot SRB2Kart Server's admin password.
Note: the password is only worth for the current session. Each time the server restarts, the password changes.

/kart_server

The /kart_server commands are for managing directly the Strashbot SRB2Kart Server. Used to start, stop or restart the server.

/kart_server start

Call the /kart_server start subcommand to start the SRB2Kart. Only works if the server isn't already running.

/kart_server stop

Call the /kart_server stop subcommand to stop the SRB2Kart. Only works if the server is running.

Options:

  • force - pass this option to True, to force the server's halt. If not, the command will fail if there are players currently on the SRB2Kart server.

/kart_server restart

Call the /kart_server stop subcommand to stop the SRB2Kart.

Options:

  • force - pass this option to True, to force the server's restart. If not, the command will fail if there are players currently on the SRB2Kart server.

/kart_addons_manage

The /kart_server commands are for managing the addons that are installed on the StrashBot Karting SRB2Kart Server.
Note that adding or removing addons from the server only take effects after the next server restart (see the /kart_server commands)

/kart_addons_manage get_order

Call the /kart_addons_manage get_order subcommand to fetch a copy of the file that controls the order in which the addons are loaded on the Strashbot Karting SRB2Kart server.

/kart_addons_manage set_order

Use the /kart_addons_manage get_order subcommand to change the config file that controls the order in which the addons are loaded on the Strashbot Karting SRB2Kart server.

Options:

  • order_config_file [Required]- the order config file to upload.

/kart_addons_manage upload_new

Call the /kart_addons_manage upload_new subcommand to upload and install a new addon on the StrashBot Karting SRB2Kart server.
Note: only *.pk3 and *.wad files are accepted.

Options:

  • kart_addon_file [Required]- the addon file to upload and install.

/kart_addons_manage link_new

Call the /kart_addons_manage link_new subcommand to upload and install a new addon on the StrashBot Karting SRB2Kart server through a direct url.
Note: only direct url to a file of type *.pk3 or *.wad are accepted.

Options:

  • addon_url [Required]- the direct url to the addon file.

/kart_addons_manage remove

Call the /kart_addons_manage link_new subcommand to uninstall and remove an addon from the StrashBot Karting SRB2Kart server.

Options:

  • addon_name [Required]- the complete addons filename (extension included) to remove.

Smash Slash Commands [All Users]

/player

The /player commands are used to register infos about Smash players in the Discord guild.
The collected data are later on used by Tournament Organiser for organisation purposes. For instance generate Weekly or Monthy tournament results graph.

/player tag

The /player tag subcommand is used to register a player's tagname tourney information into the guild's DataBase.

Options:

  • name [Required] - Register your Smash player name
  • team - Register your team's name

/player roster

The /player roster subcommand is used to register a player's character roster into the guild's DataBase, up to 4 characters

Options:

  • fighter1 [Required] - The player's "main" character's name
  • fighter2 - The player's second character's name
  • fighter3 - The player's third character's name
  • fighter4 - The player's fourth character's name
The file used to reference all the Smash Bros. figthers can be found here.

/player delete

The /player delete subcommand is remove a player's information from the guild's DataBase. Both one's character roster and tag.

/player about

Use the /player about subcommand to check your character and tag name as they are currently registered in the database. You can check another player's info by using the 'user' command option.

Options:

  • user - Mention to the user you wish to ckech info of.

Smash Slash Commands [Restricted access]

/top8

The /top8 commands are used by Smash Tournament Organizer to generate automatically result graphs for weekly or monthly tournaments.
The data used is pulled from the start.gg API and from the guild's DataBase that the player fed using the /player slash commands.

/top8 test

The /top8 test subcommand is used to check data availability from both a start.gg completed tournament, and the guild's DataBase.
When the data is processed, the Strashbot will provide the caller with a .json file representing the fetched top8 result. The caller should then use this file to pinpoint missing data. Afterwards it's up to the caller to fill in the blank of missing data with 2 methods available:

  • using the inline options option (inherited from old style commands)
  • providing a completed .json file (it is advised to check the json validity with a validator such as jsonlint.com beforehand)

Options:

  • startgg_url [Required] - the url to the completed start.gg tourney you wish to check the available data for.
  • title - the title for your data set
  • config - a .json file to provide the missing data. The json document should be construct in a the same way as the json document returned by a first call to /top8 test. The intent here is to proceed like this:
    1. run a first /top8 test
    2. fetch the returned json document and identify missing data
    3. edit this json document and fill in the missing data
    4. make sure your document is valid using a validator such as jsonlint.com
    5. re-run /top8 test with your completed json file through the config option
    6. if all is right, you can use the same json document you just provided to fill in the missing data for the /top8 draw call.
  • inline_options - old style inline options to provide missing data (e.g: ?top5a-char1="Mario 3"); check available list below.

/top8 templates

List the available templates for graph generation (using /top8 draw).

/top8 draw

The /top8 draw subcommand, provided with a template and start.gg data, uses the guild's DataBase of players character roster and tagnames (see /player) to generate a top8 result graph for a given start.gg tourney.

You probably shouldn't use this command right away as a data might be missing from the guild's DataBase. First run a few rounds of /top8 test to identify missing data and determine how to fill in the blanks.

Options:

  • startgg_url [Required] - the url to the completed start.gg tourney you wish to check the available data for.
  • title [Required] - the title for your graph
  • template [Required] - the graph's template (list available through /top8 templates)
  • config - a .json file to provide the missing data. The json document should be construct in a the same way as the json document returned by a first call to /top8 test.
  • inline_options - old style inline options to provide missing data (e.g: ?top5a-char1="Mario 3"); check available list above.
  • post_channel - a mention to the guild's channel where you want the generated graph to be posted
